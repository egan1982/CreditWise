# 策略规则挖掘工作流说明文档

## 业务场景

**适用领域**：风控/反欺诈场景下的策略规则挖掘与效果评估

**核心目标**：从历史数据中自动挖掘高效的风控规则，并通过贪心算法筛选出最优规则集，实现对坏样本的精准识别。

**典型应用**：
- 信贷审批规则挖掘
- 反欺诈策略生成
- 风险预警规则优化

---

## 规则挖掘模式

DeepAnalyze 支持两种规则挖掘模式，满足不同业务场景需求：

| 模式 | 规则类型 | 算法 | 适用场景 |
|------|----------|------|----------|
| **🎯 单特征规则** | `age <= 25` | 阈值分箱 | 简单直观、易解释、快速筛选 |
| **🌲 多特征组合规则** | `(age <= 25) & (income <= 5000)` | 决策树 | 复杂模式、交叉特征、高精度 |

### 模式对比

| 维度 | 单特征规则 | 多特征组合规则 |
|------|------------|----------------|
| **规则复杂度** | 低（单变量阈值） | 高（多变量组合） |
| **可解释性** | 强（直观易懂） | 中（需要理解组合逻辑） |
| **计算速度** | 快 | 慢（组合爆炸） |
| **规则数量** | 少（~百条） | 多（~万条） |
| **适用数据** | 特征独立性强 | 特征存在交互效应 |
| **典型用途** | 初筛规则、单变量策略 | 精细化规则、组合策略 |

> 📌 **选择建议**：
> - 初次探索数据时，先用**单特征规则**快速了解各变量的区分能力
> - 需要更精细的规则时，使用**多特征组合规则**挖掘变量间的交互效应

**规则挖掘6阶段说明**（v2.0 合并优化）：

| 阶段 | ID | 名称 | 说明 |
|------|-----|------|------|
| 1 | preprocessing | 数据预处理 | 特征名映射、删除无用列、基础清洗 |
| 2 | feature_engineering | 特征工程（可选） | 缺失值处理、IV计算、One-Hot编码 |
| 3 | generating_rules | 规则生成 | 决策树/阈值分箱生成候选规则 |
| 4 | rule_filtering | 规则筛选 | 方向过滤 + 指标评估 + 阈值筛选（合并原filtering_rules和evaluating_rules） |
| 5 | selecting_rules | 最优选择 | 贪心算法选择最优规则集，支持多目标约束（召回率、精准度、提升度） |
| 6 | report_generation | 报告生成 | 汇总结果、生成可视化报告、**Excel报告导出**、**先验规则分析**、**金额维度分析** |

**专家模式下的界面交互**：

1. **阶段执行完成**：系统自动暂停，显示当前阶段的输出结果
2. **查看阶段输出**：右侧面板展示阶段代码和执行结果
3. **🤖 AI 智能分析**（自动触发）：系统调用 LLM 分析阶段输出，生成参数优化建议（SUGGESTED_PARAMS），以建议卡片形式展示在阶段输出上方
4. **💡 AI 建议操作**：可选择「仅填入」（将建议参数填入配置面板）或「填入并重试」（填入参数后自动从当前阶段重新执行，已完成阶段结果缓存复用，无需重跑）
5. **📸 版本快照**：每次重试自动保存历史版本（参数+输出+AI分析），可在顶部版本选择器中切换对比 v1/v2/v3
6. **确认继续**：点击「继续执行」按钮进入下一阶段

**规则挖掘模式说明**：
- **单特征规则**：生成 `(age <= 25)` 形式的简单规则，基于阈值分箱
- **多特征组合规则**：生成 `(age <= 25) & (income <= 5000)` 形式的组合规则，基于决策树

**分箱方式说明**（单特征模式）：
| 方式 | 类型 | 说明 |
|------|------|------|
| **等频分箱** | 无监督 | 每个分箱包含相同数量的样本 |
| **等距分箱** | 无监督 | 每个分箱的数值范围相等 |
| **卡方分箱** | 有监督 | 基于卡方检验合并相似区间，保留区分度 |
| **决策树最佳分箱** | 有监督 | 使用决策树自动寻找最优分割点 |
| **自定义阈值** | 手动 | 用户为特定特征指定分箱阈值 |

> 📌 **选择建议**：初次探索用等频分箱；追求最佳区分度用卡方或决策树分箱；有业务经验时用自定义阈值。

## 阶段详解

### 阶段0a：数据预处理

> 对原始数据进行基础清洗和转换，为后续规则挖掘做准备。

**处理步骤**：

| 步骤 | 操作 | 说明 | 输出 |
|------|------|------|------|
| 0a.1 | 特征名映射 | 将匿名特征名(f0, f1...)映射为业务特征名 | 重命名后的列 |
| 0a.2 | 智能检测非建模列 | 自动检测ID列、时间列、样本类型列、高基数列（只标记不删除） | 自动排除报告 |
| 0a.3 | 检测常量列 | 检测只有单一值的无效列（只标记不删除） | 常量列列表 |
| 0a.4 | 日期时间处理 | 提取日期时间特征（年、月、日、星期、小时、距今天数等） | 日期衍生特征（`_dt_`标识） |
| 0a.5 | 文本列处理 | 提取文本特征（长度、词数、关键词匹配等） | 文本衍生特征（`_txt_`标识） |

**关键参数**：

**文本特征说明**：

| 特征 | 说明 | 示例 |

**使用场景**：

### 阶段1：候选规则生成

**目的**：根据选择的挖掘模式生成候选规则

---

#### 🎯 模式A：单特征规则生成

**核心算法**：
1. 对每个特征进行分箱，获取阈值点
2. 对每个阈值生成 `<=` 和 `>` 两个方向的规则
3. 汇总所有单特征规则

**关键参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `n_bins` | 10 | 分箱数量 |
| `bin_method` | 'quantile' | 分箱方式：'quantile'(等频)、'uniform'(等距)、'custom'(自定义) |
| `rule_directions` | 'both' | 规则方向：'<='、'>'、'both'(双向) |

**规则格式示例**：
```python
# 单特征规则示例
"(age <= 25)"
"(income > 50000)"
"(score <= 600)"

# 规则DataFrame结构
| used_var | rule           | threshold | direction |
|----------|----------------|-----------|-----------|
| age      | (age <= 25)    | 25        | <=        |
| age      | (age > 25)     | 25        | >         |
| income   | (income <= 5000)| 5000     | <=        |
```

**输出规模**：
- 典型场景：50个特征 × 10个阈值 × 2个方向 ≈ **1000条候选规则**

---

#### 🌲 模式B：多特征组合规则生成

**核心算法**：
1. 对所有特征进行N变量组合（默认N=3）
2. 对每个组合构建决策树
3. 从决策树叶节点向上回溯，提取规则路径

**关键参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `n_vars` | 3 | 每个组合包含的变量数 |
| `max_depth` | 5 | 决策树最大深度 |
| `min_samples_leaf` | 0.01 | 叶节点最小样本比例 |
| `max_onehot_vars` | 2 | 每个组合最多包含的One-Hot变量数 |

**规则格式示例**：
```python
# 多特征组合规则示例
"(age <= 25.5) & (income <= 5000.0) & (channel_is_online > 0.5)"

# 规则DataFrame结构
| used_var                        | rule                                    |
|---------------------------------|-----------------------------------------|
| age & income & channel_is_online| (age <= 25.5) & (income <= 5000.0) ... |
```

**输出规模**：
- 典型场景：50个特征 × C(50,3)组合 × 每组合约10条规则 ≈ **10-15万条候选规则**

---

### 阶段2：规则过滤

**目的**：根据特征分裂方向和有效性过滤无效规则

#### 3.1 确定特征分裂方向

对每个特征单独构建决策树，确定其"风险方向"：

```
特征分裂方向示例：
┌──────────────────┬───────────┬──────────┬────────────────────────┐
│ 特征             │ 切分点    │ 方向     │ 含义                   │
├──────────────────┼───────────┼──────────┼────────────────────────┤
│ age              │ 25.5      │ <=       │ 年龄≤25.5时风险高      │
│ income           │ 8000.0    │ <=       │ 收入≤8000时风险高      │
│ score            │ 650.0     │ >        │ 评分>650时风险高（异常）│
│ channel_is_A     │ == 1      │ ==       │ 二值型特征，使用等于方向│
│ desc_txt_has_急  │ == 1      │ ==       │ 文本关键词特征，二值型  │
└──────────────────┴───────────┴──────────┴────────────────────────┘
```

> 📌 **二值型特征自动识别**：
> - 系统自动识别二值型特征（0/1值），包括：
>   - One-Hot编码特征（含 `_is_` 指示符）
>   - 文本关键词特征（含 `_txt_has_` 指示符）
>   - 其他只有0/1两个值的特征
> - 二值型特征使用 `==` 方向（等于1），而非 `<=`/`>` 方向
> - 这避免了对0/1特征生成无意义的比较规则

#### 3.2 过滤规则

**过滤条件**：
1. **方向一致性**：规则中特征的方向需与分裂方向一致
2. **二值型特征**：自动使用 `==` 方向，跳过方向推断
3. **有效性检查**：排除切分点为`(-inf, inf]`的无效特征
4. **特殊变量约束**：评分类变量强制指定方向（如score必须`>`）

**过滤效果**：
- 过滤前：~10万条规则
- 过滤后：~1万条规则（约90%被过滤）

---

### 阶段3：规则效果评估

**目的**：计算每条规则的核心指标

**评估指标**：

| 指标 | 计算公式 | 业务含义 |
|------|----------|----------|
| **recall** | 规则命中的坏样本数 / 总坏样本数 | 规则对坏样本的召回能力 |
| **bad_rate** | 规则命中样本中的坏样本占比 | 规则的精准度 |
| **lift** | bad_rate / 整体bad_rate | 规则的提升倍数 |
| **hit_rate** | 规则命中样本数 / 总样本数 | 规则的覆盖范围 |

**指标阈值过滤**（默认值）：

| 条件 | 阈值 | 说明 |
|------|------|------|
| hit_rate | < 3% | 避免规则覆盖范围过大 |
| lift | > 3.5 | 确保规则有足够的提升效果 |

**输出示例**：
```
| rule                              | recall | bad_rate | lift | hit_rate |
|-----------------------------------|--------|----------|------|----------|
| (age<=25) & (income<=5000) & ...  | 0.0234 | 0.1523   | 4.2  | 0.0156   |
| (channel_is_A>0.5) & (days<=30)...| 0.0189 | 0.1892   | 5.2  | 0.0098   |
```

---

### 阶段4：最优规则集选择

**目的**：使用贪心算法选择最优规则组合

**贪心算法流程**：

```
┌─────────────────────────────────────────────────────────────────┐
│                      贪心选择算法                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 初始化：剩余样本集 = 全部样本，已选规则集 = 空              │
│                         ▼                                       │
│  2. 在剩余样本上计算所有规则的bad_rate                          │
│                         ▼                                       │
│  3. 选择bad_rate最高的规则加入已选规则集                        │
│                         ▼                                       │
│  4. 从剩余样本中移除被该规则命中的样本                          │
│                         ▼                                       │
│  5. 检查累计hit_rate是否超过阈值（默认10%）                     │
│         │                                                       │
│    是 ──┼──▶ 结束，输出最优规则集                               │
│         │                                                       │
│    否 ──┼──▶ 返回步骤2继续迭代                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**输出结果**：

| 字段 | 说明 |
|------|------|
| rule | 规则表达式 |
| used_var | 规则使用的变量 |
| lift | 单条规则的lift |
| dev_cum_recall | 累计召回率 |
| dev_cum_bad_rate | 累计坏账率 |
| dev_cum_lift | 累计lift |
| dev_cum_hit_rate | 累计命中率 |

**最终输出示例**：
```
| rule                    | lift | dev_cum_recall | dev_cum_hit_rate |
|-------------------------|------|----------------|------------------|
| (age<=25) & (inc<=5k)   | 5.2  | 0.0234         | 0.0156           |
| (channel_A) & (days<=30)| 4.8  | 0.0412         | 0.0298           |
| (region_is_X) & ...     | 4.5  | 0.0589         | 0.0445           |
| ...                     | ...  | ...            | ...              |
| 最后一条规则            | 3.6  | 0.1523         | 0.0998           |
```

## 关键参数配置指南

### 规则生成参数

| 参数 | 推荐范围 | 调优建议 |
|------|----------|----------|
| `n_vars` | 2-4 | 变量越多规则越精细，但组合数指数增长 |
| `max_depth` | 3-6 | 深度越大规则越复杂，过拟合风险增加 |
| `min_samples_leaf` | 0.005-0.02 | 越小规则越细，但可能不稳定 |

### 过滤参数

| 参数 | 推荐范围 | 调优建议 |
|------|----------|----------|
| `max_hit_rate` | 0.01-0.05 | 根据业务可接受的拒绝率调整 |
| `min_lift` | 2.0-5.0 | 根据整体bad_rate水平调整 |

### 选择参数

| 参数 | 推荐范围 | 调优建议 |
|------|----------|----------|
| `max_hit_rate_select` | 0.05-0.15 | 最终规则集的最大覆盖率 |

## 使用示例

### 基础用法：单特征规则挖掘

```python
from deepanalyze.analysis.task_SOP.rule_mining import SingleVarRuleMiner, RuleEvaluator, RuleSelector

# 准备数据
df = pd.read_csv('your_data.csv')
feature_cols = ['age', 'income', 'score', 'days']
target_col = 'target'
weight_col = 'weight'

# 1. 生成单特征规则
miner = SingleVarRuleMiner(
    n_bins=10,           # 分箱数量
    bin_method='quantile', # 等频分箱
    directions='both'    # 生成 <= 和 > 两个方向
)
rules = miner.generate_rules(df, feature_cols, target_col, weight_col)
print(f"生成规则数: {len(rules)}")

# 2. 评估规则
evaluator = RuleEvaluator()
evaluated = evaluator.evaluate_rules(df, rules, target_col, weight_col)
evaluated = evaluator.filter_by_metrics(evaluated, max_hit_rate=0.03, min_lift=3.5)

# 3. 选择最优规则集
selector = RuleSelector()
optimal = selector.select_optimal_rules(df, evaluated, target_col, weight_col)

print(optimal)
```

### 基础用法：多特征组合规则挖掘（决策树）

```python
from deepanalyze.analysis.task_SOP.rule_mining import RuleMiner, RuleEvaluator, RuleSelector

# 准备数据（已完成特征工程）
df = pd.read_csv('your_data.csv')
feature_cols = ['age', 'income', 'channel_is_A', 'channel_is_B', ...]
target_col = 'target'
weight_col = 'weight'

# 1. 生成多特征组合规则
miner = RuleMiner(n_vars=3, max_depth=5)
rules = miner.generate_rules(df, feature_cols, target_col, weight_col)

# 2. 获取分裂方向
direction = miner.get_split_direction(df, target_col, weight_col)

# 3. 过滤规则
filtered_rules = miner.filter_rules(rules, direction)

# 4. 评估规则
evaluator = RuleEvaluator()
evaluated = evaluator.evaluate_rules(df, filtered_rules, target_col, weight_col)
evaluated = evaluator.filter_by_metrics(evaluated, max_hit_rate=0.03, min_lift=3.5)

# 5. 选择最优规则集
selector = RuleSelector()
optimal = selector.select_optimal_rules(df, evaluated, target_col, weight_col)

print(optimal)
```

## 注意事项

1. **数据质量**：确保输入数据无空值，或在预处理阶段妥善处理
2. **特征数量**：特征过多时组合数爆炸，建议先做特征筛选（<100个特征）
3. **样本量**：建议样本量>10万，确保规则统计意义
4. **过拟合风险**：在验证集上评估规则效果，避免过拟合
5. **业务解释性**：最终规则需业务专家审核，确保逻辑合理

### 1. 先验规则评估 (Prior Rules Evaluation)

**功能说明**：在效果评估阶段，支持传入已有的先验规则集，分析新规则的增量贡献。

**应用场景**：
- 已有一套线上运行的风控规则，需要评估新规则的边际效果
- 避免新规则与现有规则高度重叠，造成资源浪费
- 量化新规则能额外捕获多少坏样本

**输出指标**：
| 指标 | 说明 | 计算方式 |
|------|------|----------|
| **standalone_recall** | 规则独立召回率 | 规则命中的坏样本 / 总坏样本 |
| **incremental_recall** | 增量召回率 | 在先验规则基础上额外捕获的坏样本 / 总坏样本 |
| **overlap_rate** | 重叠率 | 与先验规则重叠命中的样本 / 规则命中样本 |
| **marginal_contribution** | 边际贡献 | incremental_recall / standalone_recall |

**参数配置**：
```python
# API模式
pipeline = RuleMiningPipeline()
results = pipeline.run(
    df=df,
    target_col='target',
    prior_rules=[  # 先验规则列表
        "(age <= 25) & (income <= 5000)",
        "credit_score <= 550"
    ]
)
```

**WebUI配置**：在参数面板的「高级参数」中配置先验规则列表。

### 2. 金额维度分析 (Amount Analysis)

**功能说明**：在效果评估阶段，支持传入金额列，分析规则在金额维度的表现。

**应用场景**：
- 风控场景中，不仅关注命中率，还关注命中金额
- 评估规则对高风险金额的覆盖能力
- 计算金额加权的lift指标

**输出指标**：
| 指标 | 说明 | 计算方式 |
|------|------|----------|
| **hit_amount** | 命中金额 | 规则命中样本的金额总和 |
| **bad_amount** | 坏样本金额 | 规则命中的坏样本金额总和 |
| **amount_recall** | 金额召回率 | 命中的坏样本金额 / 总坏样本金额 |
| **amount_lift** | 金额提升度 | amount_recall / (hit_amount / 总金额) |
| **avg_bad_amount** | 平均坏样本金额 | bad_amount / 命中坏样本数 |

**参数配置**：
```python
# API模式
pipeline = RuleMiningPipeline()
results = pipeline.run(
    df=df,
    target_col='target',
    amount_col='loan_amount'  # 金额列名
)
```

**WebUI配置**：在参数面板选择「金额列」下拉框。

**WebUI展示**：在结果面板的「金额分析」Tab中查看完整金额维度指标。
