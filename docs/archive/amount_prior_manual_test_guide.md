## 0. 业务逻辑说明

### 0.1 金额维度分析（P2-10）

规则挖掘默认只从**人数维度**评估规则效果（命中率、召回率、Lift）。但在实际风控中，不同客户的**损失金额差异巨大**——一条规则可能只命中 5% 的坏客户，但这 5% 客户贡献了 30% 的损失金额。

**金额分析的核心指标**：

| 指标 | 公式 | 业务含义 |
|------|------|---------|
| 命中金额 | 规则命中样本的金额之和 | 规则拦截了多少钱 |
| 金额占比 | 命中金额 / 总金额 | 拦截覆盖面 |
| 金额坏账率 | 坏账金额 / 命中金额 | 命中客户中坏账的金额集中度 |
| 金额 Lift | 金额坏账率 / 整体金额坏账率 | 规则在金额维度的区分能力 |
| 金额累计召回率 | 累计坏账金额 / 总坏账金额 | 规则集拦截了多少比例的损失 |

**使用方式**：参数面板 → 高级参数 → 报告生成 → 「损失金额列（可选）」下拉框选择金额列。

**已知问题（FIX-1~FIX-6）**：之前单元测试发现全栈数据流断裂等 6 个 Bug，文档标注已修复，本次人工测试同时作为**回归验证**。

### 0.2 先验规则分析（P2-7）

用户可能已有一套生产中的风控规则。先验规则分析功能允许用户输入这些旧规则，与新挖掘的最优规则进行**对比分析**。

**分析指标**：

| 指标 | 业务含义 |
|------|---------|
| 独立召回率 | 先验规则单独能拦截多少坏客户 |
| 增量召回率 | 在最优规则基础上，先验规则额外拦截了多少 |
| 重叠率 | 先验规则与最优规则的命中样本重叠程度 |
| 边际贡献 | 先验规则对总召回的增量贡献 |

**三种输入方式**：
1. **手动输入**：textarea 中每行一条表达式（如 `(f47 > 100)`）
2. **上传 CSV**：结构化格式或表达式格式
3. **从工作区选择 CSV**

**三级校验**：
- L1 硬拒：非 CSV / 0 条规则 → 红色错误
- L2 警告：部分列名不存在 → 黄色警告
- L3 通过：全部匹配 → 绿色提示

## 4. 关键验证点速查

### 4.1 金额分析前端数据流

```
Pipeline results['amount_analysis'] (扁平化 dict)
  → safe_serialize → API response
  → 前端 unwrapData(outputs.amount_analysis)
  → AdvancedAnalysisPanel → AmountAnalysisPanel(analysis={...})
```

FIX-1 扁平化后结构：
```json
{
  "enabled": true,
  "amount_col": "loss_amount",
  "total_amount": 123456.78,
  "total_bad_amount": 45678.90,
  "overall_amount_bad_rate": 0.37,
  "cumulative": { "cum_hit_amount": ..., "cum_bad_amount": ..., "amount_recall": ..., "cum_amount_bad_rate": ..., "cum_amount_lift": ... },
  "rules_amount": [ { "rule": "...", "hit_amount": ..., "amount_lift": ... }, ... ]
}
```

### 4.2 先验规则前端数据流

```
Pipeline results['prior_analysis'] (扁平化 dict)
  → safe_serialize → API response
  → 前端 unwrapData(outputs.prior_analysis)
  → AdvancedAnalysisPanel → PriorAnalysisPanel(priorAnalysis={...})
```

### 4.3 参数位置

| 参数 | 面板位置 | 类型 |
|------|---------|------|
| `amount_col` | 高级参数 → 报告生成 → 损失金额列 | column_combobox（下拉选择列名） |
| `prior_rules` | 高级参数 → 报告生成 → 先验规则 | prior_rules_input（三 Tab 输入） |

### 4.4 CSV 模板格式

**表达式格式**:
```csv
rule_id,expression
R001,(f47 > 100)
R002,(f73 > 50)
```

**结构化格式**:
```csv
rule_id,rule_name,feature,operator,threshold,direction
R001,高风险,f47,>,100,reject
R002,高负债,f73,>,50,reject
```
