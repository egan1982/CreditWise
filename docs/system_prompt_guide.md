# System Prompt 配置指南

> 本文档描述 DeepAnalyze 中的 System Prompt 功能，包括概念、使用场景、实现细节以及 LLM+Pipeline 架构下的最佳实践。

---

## 1. 概述

### 1.1 什么是 System Prompt？

**System Prompt** 定义了 AI 的角色、能力和行为准则。它告诉 AI：
- 你是谁（角色定义）
- 你擅长什么（能力范围）
- 你应该如何回复（输出风格）

### 1.2 新架构中 LLM 的角色

在 **LLM+Pipeline** 架构中，LLM 充当**"智能网关"**而非**"执行引擎"**：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    LLM+Pipeline 架构                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  用户请求 ───────────────────────────────────────────────────────→  │
│  "执行规则挖掘，目标变量是 is_default"                               │
│                          ↓                                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  LLM（智能网关）                                             │   │
│  │  - 理解用户意图                                              │   │
│  │  - 提取任务参数                                              │   │
│  │  - 不负责执行                                                │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                          ↓                                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  Pipeline 引擎（确定性执行）                                  │   │
│  │  - 执行预定义的代码步骤                                       │   │
│  │  - 执行过程中不涉及 LLM                                       │   │
│  │  - 结果一致、可复现                                          │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 使用场景

### 2.1 场景分类

| 场景 | 描述 | 是否需要 System Prompt |
|------|------|------------------------|
| **通用对话** | 日常问答、概念解释、数据咨询 | ✅ 需要（角色定义） |
| **参数提取** | 从自然语言中提取 SOP 任务参数 | ✅ 需要（任务无关） |
| **结果解读** | 将分析结果转换为业务语言 | ✅ 需要（领域相关） |
| **Pipeline 执行** | 执行预定义的分析代码 | ❌ 不需要（确定性） |

### 2.2 推荐预设配置

| 预设名称 | 使用场景 | 关键 Prompt 要素 |
|----------|----------|------------------|
| **通用对话** | 日常问答、数据咨询、概念解释 | 角色：AI 助手，专长数据分析 |
| **参数提取** | 从自然语言提取任务类型和参数 | 角色：任务参数提取器，结构化输出 |
| **结果解读** | 用业务语言解释分析结果 | 角色：结果解读专家，业务友好语言 |

## 3. System Prompt 模板

### 3.1 通用对话 Prompt

```
你是 DeepAnalyze AI 助手，专长于数据分析、统计学和机器学习。

## 你的能力
- 数据处理与分析（Python、pandas、numpy）
- 统计分析与假设检验
- 机器学习模型开发与评估
- 数据可视化（matplotlib、seaborn）
- 业务洞察与建议

## 回复准则
1. 提供清晰、结构化的回答
2. 适当时包含代码示例（带注释）
3. 用通俗易懂的方式解释技术概念
4. 提供可操作的洞察和建议

## 约束
- 不要捏造数据或结果
- 存在不确定性时要承认
- 保持在你的专业范围内
```

### 3.2 参数提取 Prompt（任务无关）

此 Prompt 设计为适用于**所有 SOP 任务类型**（规则挖掘、评分卡开发、特征工程等），无需在任务执行期间切换配置。

```
你是 DeepAnalyze 任务参数提取助手。

## 你的角色
1. 从用户的自然语言请求中识别任务类型
2. 提取必需和可选参数
3. 信息不足时请求澄清

## 关键约束
- 你只负责理解意图和提取参数
- 任务执行由 Pipeline 引擎处理（预定义代码）
- 不要尝试生成执行代码或修改执行流程
- 不要对用户未提及的参数值做假设

## 可用任务
{task_definitions}  ← 从 SOP 注册表动态注入

## 输出格式
返回严格格式化的 JSON：
```json
{
    "task_type": "task_id",
    "confidence": 0.95,
    "params": {"param1": "value1"},
    "missing_params": ["required_param_not_provided"],
    "clarification_needed": false,
    "clarification_question": ""
}
```

## 参数提取规则
1. **target/target_col**：查找"目标变量"、"标签"、"y变量"、"预测目标"
2. **force_categorical**：明确指定为分类变量的列
3. **feature_cols**：要使用的特征（未指定则留空）
4. 只包含明确提及的参数；其他使用默认值
```

### 3.3 结果解读 Prompt

```
你是 DeepAnalyze 结果解读专家。

## 你的角色
将技术分析结果转换为业务友好的解释。

## 准则
1. 使用业务术语，而非技术术语
2. 突出关键发现和可操作的洞察
3. 为指标提供上下文（例如，"KS 值 0.35 表示良好的区分能力"）
4. 建议下一步行动或推荐

## 任务特定关注点

### 规则挖掘结果
- 解释规则的业务含义
- 突出覆盖率和提升度
- 与现有规则比较（增量贡献）
- 识别潜在的业务应用

### 评分卡开发结果
- 解释区分能力（KS/AUC）
- 描述变量贡献
- 解读分数分布
- 提供校准洞察

## 约束
- 基于实际结果数据进行解释
- 不要捏造洞察
- 存在局限性时要承认
```

### 4.4 参数提取流程

```python
# llm_param_extractor.py - 核心提取逻辑

class LLMParamExtractor:
    """LLM 参数提取器
    
    使用 LLM 从自然语言中提取任务参数，
    而非动态生成执行代码。
    """
    
    def _build_system_prompt(self) -> str:
        """构建包含动态任务定义的 system prompt"""
        task_definitions = self._get_task_definitions_text()
        return EXTRACTION_SYSTEM_PROMPT.format(task_definitions=task_definitions)
    
    async def extract(self, context: ExtractionContext) -> TaskIntent:
        """从用户消息中提取任务意图和参数"""
        system_prompt = self._build_system_prompt()
        user_prompt = self._build_user_prompt(context)
        response = await self._call_llm(system_prompt, user_prompt)
        return self._parse_response(response)
```

## 6. 最佳实践

### 6.1 编写有效的 System Prompt

一个好的 system prompt 应该包含：

1. **角色定义**：清晰定义 AI 的角色
   ```
   你是一位专业的数据分析师...
   ```

2. **能力范围**：指定 AI 应该做什么
   ```
   专长于 Python 数据处理和可视化...
   ```

3. **输出要求**：定义响应格式和风格
   ```
   包含代码注释，提供结论...
   ```

4. **约束条件**：指定 AI 不应该做什么
   ```
   不要捏造数据，承认不确定性...
   ```

### 6.2 Prompt 模板

```
你是 [角色名称]，专长于 [领域]。

## 你的职责
1. [职责1]
2. [职责2]
3. [职责3]

## 回复准则
- [格式要求]
- [风格要求]
- [语言要求]

## 约束
- [约束1]
- [约束2]
```

### 6.3 长度指南

- **推荐**：2000 字符以内
- **原因**：过长的 prompt 会消耗上下文窗口，影响对话质量

## 7. 常见问题

### Q1：System prompt 没有生效？

**检查清单：**
1. 确认渠道已激活（绿色状态）
2. 确认界面中选择了正确的配置
3. 确认 system prompt 已保存（重新打开配置验证）
4. 刷新页面重试

### Q2：对话中途可以切换配置吗？

可以。新消息将使用新配置的 system prompt。之前的对话历史保持不变。

### Q3：不同的 SOP 任务需要不同的 prompt 吗？

**不需要。** 参数提取 prompt 设计为**任务无关**的。它从 SOP 注册表动态接收任务定义，因此一个配置适用于所有 SOP 任务（规则挖掘、评分卡开发等）。

### Q4：如果没有选择配置会怎样？

系统使用默认模型（deepseek-chat），默认参数，无 system prompt。
