## 一、项目概述

### 1.1 项目定位

CreditWise 是一款 **AI 驱动的信贷风控智能分析平台**，专注于将信贷业务中最核心、最耗时的两项分析工作 —— **策略规则挖掘**和**评分卡建模** —— 从传统的"专家手工操作"升级为"AI 自动化 + 人工审核"模式。

| 维度 | 说明 |
|------|------|
| **业务领域** | 信贷风控（贷前审批、贷中监控、贷后管理） |
| **核心用户** | 风控策略分析师、数据科学家、风控团队管理者 |
| **核心价值** | 降低风控建模技术门槛，缩短分析周期，提升策略迭代效率 |
| **产品形态** | Web 应用（浏览器访问），支持私有化部署 |

### 1.2 业务痛点与解决方案

| 业务痛点 | 传统方式 | CreditWise 方案 |
|---------|---------|----------------|
| 规则挖掘耗时长 | 分析师手工探索特征、编写代码、反复调参，通常需要 **2-5 天** | 上传数据 → 配置参数 → **自动完成全流程**（数据预处理 → 特征筛选 → 规则生成 → 规则评估 → 最优规则选择），**分钟级**完成 |
| 评分卡开发门槛高 | 需要统计学背景、掌握 WOE/IV/逻辑回归等专业知识 | **标准化 Pipeline** 自动完成分箱、WOE 转换、特征筛选、建模、评分转换、模型评估全流程，输出专业评分卡报告 |
| 分析结果不可复现 | 手工分析过程难以记录和复现 | 每次任务执行全程记录，支持**暂停/恢复/重试**，结果可追溯 |
| LLM 配置管理混乱 | 各团队自行管理 API Key，缺乏统一管控 | 内置 **LLM Manager**，统一管理多供应商渠道，支持负载均衡、调用日志、成本统计 |
| 部署与数据安全 | 公有云 SaaS 不满足金融行业数据合规要求 | 支持**内网私有化部署**，数据不出网，Docker 一键部署 |

CreditWise 并非简单的 fork，而是在 DeepAnalyze 的通用数据分析能力基础上，进行了**深度垂直化改造**：

```
DeepAnalyze（通用数据分析 Agent）
    │
    ├── 保留：Agentic LLM 对话引擎、代码执行沙箱、工作区文件管理
    │
    └── CreditWise 新增：
        ├── Task SOP 系统（信贷风控专用结构化工作流）
        ├── 规则挖掘 Pipeline（策略规则自动发现与评估）
        ├── 评分卡开发 Pipeline（端到端评分卡建模）
        ├── LLM Manager 深度集成（多供应商统一管理）
        ├── LLM + Pipeline 混合架构（智能入口 + 确定性执行）
        ├── 内网多用户认证部署方案
        └── 专业风控报告（Word/Excel/HTML）
```

## 四、项目亮点与差异化价值

### 4.1 架构创新：LLM + Pipeline 混合架构

这是本项目最核心的架构创新——**将 LLM 的智能性与 Pipeline 的确定性相结合**：

| 组件 | 角色 | 优势 |
|------|------|------|
| **LLM（智能入口）** | 理解自然语言意图，推断任务参数 | 降低使用门槛，用户无需了解参数细节 |
| **Pipeline（确定性执行）** | 预定义的标准化分析流程 | 执行路径可预测、结果可复现、无 LLM 幻觉风险 |

**为什么不用纯 LLM？** 在项目探索阶段，曾尝试过让 LLM 完全主导分析流程（LLM SOP 模式），但发现：
- 执行路径不可控，每次结果不一致
- LLM 调用成本高（每阶段都要调用）
- 代码安全性差（LLM 动态生成代码需要沙箱）

最终确立了当前架构：**LLM 只做"翻译官"（自然语言 → 参数），Pipeline 做"执行者"（确定性流程）**，兼顾了智能性和可靠性。

### 4.2 全流程自动化的 SOP 任务系统

将风控分析中的最佳实践固化为标准操作流程（SOP），实现了：

- **7 阶段规则挖掘 Pipeline**：从原始数据到推荐规则集，全自动执行
- **7 阶段评分卡开发 Pipeline**：从数据到可上线评分卡，全自动执行
- **任务全生命周期管理**：创建 → 执行 → 暂停 → 恢复 → 完成 → 报告导出
- **两种交互模式**：全自动模式（一键执行）+ 专家模式（阶段审核确认）

### 4.3 安全与合规

| 安全措施 | 说明 |
|---------|------|
| 内网部署 | Docker 容器化，数据不出企业网络 |
| 密码安全 | bcrypt 哈希存储，防暴力破解（5 次失败锁定 15 分钟） |
| API 密钥加密 | Fernet（AES-128-CBC）加密存储 |
| 角色权限 | admin/user 双角色，LLM 管理仅限 admin |
| 代码安全审计 | P0 级 9 项全部修复、P1 级必修项 100% 修复 |
| 路径遍历防护 | 所有文件操作端点均有路径校验 |

### 4.4 文档驱动的开发模式

项目采用"设计先行、文档驱动"的开发模式，产出了 **167 份 Markdown 文档**（总计约 1.5MB），覆盖：

## 八、项目价值总结

### 8.1 已实现的业务价值

1. **效率提升**：将规则挖掘和评分卡开发从"天级"缩短为"分钟级"，分析效率提升数十倍
2. **门槛降低**：业务人员通过自然语言对话即可触发专业风控分析，不需要编程能力
3. **标准化输出**：分析流程标准化，结果格式统一，支持 Word/Excel/HTML 多格式报告
4. **合规部署**：内网私有化部署，满足金融行业数据安全要求
5. **成本可控**：多 LLM 供应商支持，避免供应商锁定，通过负载均衡优化调用成本

### 8.2 技术资产积累

| 资产类型 | 说明 |
|---------|------|
| **可复用的 SOP 框架** | Task SOP 系统可快速扩展到其他金融分析场景 |
| **LLM+Pipeline 架构模式** | 可推广到其他需要"AI 智能 + 业务确定性"结合的场景 |
| **完整的安全审计体系** | P0/P1/P2 三级审计标准，可作为团队安全开发规范 |
| **文档驱动开发模式** | 167 份设计文档，为项目持续迭代和团队知识传承提供保障 |

### 8.3 风险与建议
